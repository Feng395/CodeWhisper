## 使用示例

### GUI 模式

启动应用后，直接语音输入：

```
语音: "这个磨具需要用慢思割一道"
识别: "这个磨具需要用慢思割一道"
修正: "这个模具需要用慢丝割一刀"
```

### 命令行模式

```bash
python app.py -f audio.wav
```

输出会显示修正详情：

```
📝 转录结果:
这个模具需要用慢丝割一刀

词典修正统计:
  总规则数: 316
  修正次数: 3

修正详情:
  1. '磨具' -> '模具' (mold)
  2. '慢思' -> '慢丝' (mold)
  3. '一道' -> '一刀' (mold)
```

## 添加新术语

### 方法 1：使用交互式工具

```bash
python tools/add_term.py
```

按提示输入术语信息即可。

### 方法 2：手动编辑字典

编辑 `dictionaries/programmer_terms.json`：

```json
{
  "categories": {
    "mold": {
      "terms": {
        "新术语": {
          "correct": "新术语",
          "description": "术语说明",
          "variants": [
            {
              "wrong": "错误形式",
              "description": "误识别原因"
            }
          ]
        }
      }
    }
  }
}
```

## 工作原理

1. **语音识别**：Whisper 将语音转为文字
2. **词典匹配**：在字典中查找可能的错误识别
3. **文本替换**：将错误形式替换为正确形式
4. **结果输出**：返回修正后的文本

详细说明请参考：[词典修正机制说明](DICTIONARY_CORRECTION.md)

## 常见问题

### Q: 为什么有些错误没有被修正？

A: 只有在字典中预定义的错误变体才会被修正。如果遇到新的错误识别，请添加到字典中。

### Q: 如何提高识别准确率？

A: 
1. 说话清晰，语速适中
2. 减少环境噪音
3. 使用质量好的麦克风
4. 持续完善字典

### Q: 修正会不会误伤正确的词？

A: 字典设计时已考虑边界匹配，避免误伤。但如果发现问题，请及时反馈。
